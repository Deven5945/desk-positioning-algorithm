# 교실 자리 교체 최적화 시스템

5행 × 6열 교실에서 현재 배치와 목표 배치 사이의 이동 순서·경로를 비교하는 브라우저 시뮬레이터입니다.

## 실행

정적 웹 페이지라서 `index.html`을 브라우저로 열거나, 프로젝트 폴더에서 간단한 로컬 웹 서버를 실행하면 됩니다.

```powershell
python -m http.server 8000
```

그 뒤 `http://localhost:8000`을 열면 됩니다.

## 모델 규칙

- 목표 배치는 Fisher–Yates 셔플로 생성합니다.
- 30명 모두 목표 자리가 현재 자리와 다릅니다.
- 1:1 맞교환 목표는 생성하지 않습니다. 이 모델에는 임시로 책상을 둘 빈 공간을 두지 않았기 때문입니다.
- 좌석 사이의 통로를 별도 이동 격자로 모델링해, 책상은 통로를 거쳐 이동합니다.
- 한 단계에서 같은 이동 칸을 차지할 수 없고, 맞바꾸기 이동은 막습니다.
- 세 칸 이상이 동시에 비워지는 순환 이동은 허용합니다.

표의 `방지한 충돌`은 실제 충돌이 일어나기 전에 스케줄러가 막은 같은-칸 경쟁 또는 맞교환 시도의 횟수입니다. 대기 시간과 혼동되지 않도록 분리해 계산합니다.

## 검증

```powershell
npm test
```

테스트는 목표 셔플의 고정 자리·맞교환 방지, A*의 고정 장애물 우회, 스케줄러의 맞교환 차단과 3개 순환 이동을 검사합니다.
